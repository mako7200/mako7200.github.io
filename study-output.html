          ■使わないコードは書かない
          (例)int jと記述したのに、コード内でjが未使用の場合

          ■コメントは以下のどちらか
          ①「該当する文字がある場合、もう一度ループさせる」（what）
          ②「同じ文字が2文字以上ある場合の対応」（why）
          → コメントは大事なことが書かれていることが多いため、ほかの人に割と読まれる。そのため、簡潔に書くことが大切。そして何よりも大切なのは、嘘は書かないということ。余分なコメントでコードが汚染されるのを防ぐため。
          → 伝わる日本語で簡潔に書く。
          ※万が一、「やりたいことの思い」を書く場合は、スクリプト最上部に記述せよ。

          ■変数の使いまわしは危険
          - 例えば、For文の中にFor文があった場合は、変数カウンタを2つ用意する必要がある。iとjというように。同様に、変数も使いまわすとバグが起きやすい例がある。

          ■DoループよりForループ
          - ループに関するカウンタ（最大値や最小値が変化する）は基本的にはノータッチ。
          → 無限ループに入りやすいため。

          ■生きたコードは最上部へ
          - 画面を開いたときに、すべてコメントアウトされたコードが目に飛び込んでくるのはよくない。今動いている生きたコードが最初に表示されるようにするべき。

          ■型変換（文字型⇔数値型）
          - String型”あ”をInteger()に変換すると、０を返す
          - なんで０になるかというと、文字と数値の二つを足す場合があるため。

        【データベース研修】


        ポイント

        - アプリが読み込まれる仕組み
        - 仕事のメモのコツ
        - 理解のコツ
        - プロトコル
        - エラー一覧
        - テーブルコントロールユニットのインストール
        - SQLの練習


        詳細内容

        ■アプリを読み込むには
        Windowsの仕組みで、同じファイルの中にあるもののみ有効。
        つまり、実行するアプリの入っているファイル内にある中身がないとだめ。
        ローカルの中身を見る。そこは有効。＋αで、パスの通っているものも読み込むよって話。
        →基礎中の基礎の知識であるため、参考書などには載っていない。こういった情報は、自分で調べて理解しておく必要がある。

        ■仕事でメモを取るコツ
        （例）以下を印刷
        登録の２得意先、５商品、１５倉庫
        設定の３区分
        →仕事の基本として、メモの取り方を洗練する必要がある。実際の現場では、メモなど取る暇もないくらいのスピードで話が進んでいくため、メモを洗練する必要がある。必要な情報のみパパっと抜き出し、メモを取る。わからない際は、確認する。
        →印刷する際は、横着せず、何枚あるのか確認し、その都度取りに行く。

        ■理解のコツ
        抽象的な意味と具体的な例のセットで覚える
        概要をつかみ、具体例を落とし込む。


        ■プロトコル
        「80」:HTTP通信のデフォルトのプロトコル
        「1521」:主にOracleデータベースが使用するポート番号


        ■テーブルコントロールユニット
        ユーザー名:PS_TERA_2024OJT
        パスワード:PS_TERA_2024OJT
        データベース:HV-ORA19CSVR:1521/RAKUSYOU
        →1521がポート番号
        →スラッシュ以降がサービス名
        ※画面が白くなったら、データベースが間違っているという意味

        ■「IPアドレス」と「ポート番号」
        - **IPアドレス:** ネットワーク上のデバイスの一意の識別子。
        - **ポート番号:** 同じデバイス上の特定のサービスやアプリケーションの識別子。
        →IPアドレスはデバイスそのものを、ポート番号はそのデバイス上の特定のサービスやアプリケーションを識別するために使われる。

        ■SQL
        - タブは使わない
        - 予約語は大文字
        - 最後に「;」セミコロン
        - エラー「無効な識別子です」は、カラム名やテーブル名のミス
        - エラー「SQL文が無効です」予約語のミス
        - ダブルで囲むとカラム名、シングルで囲むと文字列
        - エイリアスasが省略可能。（基本asなし）
        - 全角だと、カラム名が大文字で表示される
        - カラム名の後にカンマを入れ、スペースを１つ入れる
        - 外部結合はWHERE句の最後に（+）を記述するだけ
        - コメントは2種類
        - //1行コメント（--でも可）
        - /＊ブロックコメント＊/
        - BETWEEN演算子
        カラム名 BETWEEN 1 AND 10
        - IN演算子
        カラム名 IN（ 2, 4 ）
        - NOT演算子
        - １０以上でないもの
        WHERE NOT ( sok_id >= 10 )
        - IS NOT演算子
        - ２と４以外のもの
        WHERE sok_id NOT IN ( 2, 4 )
        - IS NULL/IS NOT NULL演算子
        - 倉庫住所がNULLのもの
        WHERE sok_jyu1 IS NULL
        - 倉庫住所がNULLでないもの
        WHERE sok_jyu1 IS NOT NULL
        - LIKE演算子
        - 住所に”東京”が含まれる
        WHERE sok_jyu1 LIKE '%東京%'
        - 住所に”東京”または”千葉”が含まれる
        WHERE sok_jyu1 LIKE '%東京%'
        OR sok_jyu1 LIKE '%千葉%'
        【重要】
        SQL文は、直書きしない。メモなどに書いてから、SQLに書く。
        →SQLが落ちる可能性があるため。


        本日の気づき

        - SQL上達のコツは、ひたすらSQL文を書くこと
        - 実際の現場では想像以上に進捗速度が速いため、メモを工夫して取らなければならない

        タイトル

        【SQL研修】【外部研修】


        ポイント

        - SQL文の練習
        - 「char」と「varchar」の違い
        - インラインビュー
        - ロジカル・シンキング【基礎】（外部研修）


        詳細内容

        ■「得意先検索の画面」通りのデータを抽出するSQL文を書く練習
        テーブルコントロールユニットを使用。
        WHERE句の結合がうまくできず、苦戦した。

        ■「char」と「varchar」の違い
        固定長か、可変長かの違い。文字数が決まっているキャラ、自由に設定できるバーキャラ。
        →「char」は固定長で長さが決まっている。8文字なら8文字。
        →「varchar」は可変長で、長さを自由に決めることができる。
        →Oracleでは、固定長の文字数に満たないデータはスペースで埋められる。


        ■インラインビュー
        仮想のテーブル
        →FROM句の中にカッコで記述し、エイリアス名を付けて作成
        →FROM tokuisaki (SELECT * tok_id FROM tokuisaki WHERE tok_id = '1' ) tok_1のように書く
        →コードが読みやすくなり、可読性が向上する。※ただし、可読性が低下する場合もあるので、場面によって使い分ける。時代とともに読みやすさが変わるので、柔軟に対応する。


        ■外部研修
        テーマ：『ロジカル・シンキング【基礎】』
        →ロジカルシンキングを習得することで、筋道の通った話し方ができるようになり、説得力のある話ができるようになる。
        →思考を紙に書き出して図式化することで、鍛えられる。
        →ゼロベースで考えることも大切。思考のリセット。

        本日の気づき

        - 印刷をする際は、余計な余白がないかプレビューで確認してから印刷する。「すべての列を１ページで印刷する」という指定をいれたり、ページ指定をして印刷をすることで、無駄をなくせる。

        - dual
        - DECODE関数
        - CASE句

        - NULLの出力
        - NVL関数
        - 型変換
        - 色々な関数（SUM,MIN,MAX,AVG,INSERT,SUBSTR,TRUNC,ROUND,CEIL）

        ■文字コード
        - アスキーコード

        ■トランザクション
        - 「COMMIT」と「ROLLBACK」
        - 「セッション」はデータベースに接続すること
        - 「トランザクション」と「セッション」はセットで覚える
        →「COMMIT」はトランザクションを確定すること
        →「ROLLBACK」はトランザクションの開始状態に戻すこと
        【重要】
        （★）SQLでトランザクションを実行したら、必ずrollbackかcommitしてトランザクションを終了しておく。そうしないと、待ち状態が発生してほかの人がトランザクションを実行できない。
        （★）1回トランザクションをCOMMITしたら、ROLLBACKしても元に戻せない。COMMITは超慎重に行う。必ず手打ちで「COMMIT;」と打ち込み、コミットを行う。

        ■トランザクションの詳細
        - INSERT
        - UPDATE
        - DELETE
        - SELECT INSERT
        - rownum ※プログラムでは書かない
        - SUM()
        - TO_DATE()
